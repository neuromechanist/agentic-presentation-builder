# Examples And Themes

## Shipped example decks

Use the `examples/` directory as both documentation and regression coverage.

| File | Purpose |
| --- | --- |
| `hello-world.json` | Minimal deck structure |
| `advanced-features-demo.json` | Mixed feature showcase |
| `animation-demo.json` | Fragment and animation behavior |
| `callout-demo.json` | Callout styling and tone |
| `comprehensive-demo.json` | Broad feature coverage including Mermaid |
| `github-course-demo.json` | Longer instructional flow |
| `image-demo.json` | Image sizing and multi-media layouts |
| `markdown-math-demo.json` | Markdown, alerts, code, and LaTeX |
| `theme-showcase-default.json` | Default theme |
| `theme-showcase-dark.json` | Dark theme |
| `theme-showcase-academic.json` | Academic theme |

## Theme selection

Built-in themes:

- `default`
- `light`
- `dark`
- `academic`
- `minimal`

Use `customTheme` when you need project-specific colors or font stacks.

## Media assets

Shared sample media lives in `examples/assets/`.

For custom decks:

- keep large visuals readable at full HD
- prefer percentage-based image sizing on mixed-content slides
- use `alt` text even for internal presentations, since it also improves author clarity
